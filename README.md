## Player balance management

Краткое описание работы приложения

1. При открытии приложения загружаются devices. В это время отображается loader.
2. Юзер должен выбрать игрока. Пока игрок не выбран, отображается предложение выбрать любого игрока.
3. При нажатии на кнопки Deposit или Withdraw появляется модальное окно. В этот момент в память приложения сохраняется выбранный игрок. Модальное окно единое на обе операции.
4. Перед отправкой USD происходит двухступенчатая валидация инпута. Сначала валидируется инпут "на лету", чтобы юзер мог ввести нужное значение. При сабмите значение валидируется более строго.
5. Если при нажатии Confirm значение не валидное, либо при операции withdraw не хватает денег, то будет показана соответствующая ошибка.
6. Если при нажатии Confirm все ок, оно конвертируется в KES. Отправляется запрос на сервер с обновлением значения.
7. По результатам отправки юзеру может быть показан соответствующий toast.
8. Происходит refetch данных. Чтобы произвести обновление, я запрашиваю с сервера только один device, а не все сразу.

Прочие заметки:

- настроен адаптивный дизайн
- общие константы / хелперы / типы вынесены в shared слой
